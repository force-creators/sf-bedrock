# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

Use GitHub private vulnerability reporting for this repository if it is
available. If private reporting is not available, contact the repository owner
through the maintainer contact information on their GitHub profile and include
only enough detail to establish a private reporting channel.

Helpful initial information includes:

- Affected component or feature.
- High-level impact.
- Whether the issue is reproducible in a clean Salesforce org.
- Any known safe mitigation.

Please avoid sharing exploit code, secrets, customer data, org identifiers, or
other sensitive details until a private channel is established.

## Supported Versions

Security fixes are considered for the latest public release. Pre-release code is
provided for evaluation and may change before a stable release.
